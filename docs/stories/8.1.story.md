## Context

HekiLight v0.6.0 shipped with three visual identity gaps:

1. **No AddOn list icon** â€” the `?` placeholder appeared because `HekiLight.toc` had no `## IconTexture` directive.
2. **Generic minimap icon** â€” the minimap button used `ability_whirlwind`, an unrelated spell icon with no connection to the addon.
3. **Blizzard tooltip warning** â€” hovering the minimap button showed "This is a custom button. Please ask the addon author to use the standard LibDBIcon library instead." because the `OnEnter` script called `GameTooltip:SetOwner(self, ...)` on a `Minimap`-parented button. Blizzard hooks `GameTooltip` and intercepts for any non-LibDBIcon minimap child.

The root cause of the tooltip warning turned out to be LeatrixPlus, not Blizzard directly. LeatrixPlus's `MakeButtons()` scans `Minimap:GetChildren()` and wraps every unrecognised button in a fake LibDBIcon entry that injects the warning. The fix registers HekiLight with `LibDBIcon-1.0` via `LibStub` when available, making LeatrixPlus treat it as a first-class button. A manual `Minimap`-parented button is used as fallback when LibDBIcon is not loaded.
