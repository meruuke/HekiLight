# Story 9.2: Proc Slot ÔÇö Hide When Spell Is On Cooldown

**Epic:** EPIC-9 ÔÇö Proc Slot Reliability
**Status:** Ready for Review
**Priority:** High

## User Story

As a player, I want the proc alert slot to stay hidden when the proc spell is on cooldown, so I only see procs I can actually cast right now.

## Background

`UpdateProcAlert` currently shows the proc icon for any overlay-active spell that has a keybind (Story 9.1). It does not check whether the spell is off cooldown. `IsSpellAvailable(spellID)` already exists in the codebase (~line 1300) and uses the secret-number pcall trick on `C_Spell.GetSpellCooldown`:

- `duration == 0` Ôćĺ off cooldown / has a charge Ôćĺ returns `true`
- `duration > 0` Ôćĺ on cooldown Ôćĺ pcall catches the secret-number comparison error Ôćĺ returns `false`

This function is already used by `GetSuggestionQueue` to grey out secondary slots. The proc slot should use the same gate.

## Acceptance Criteria

- [x] **AC1 ÔÇö Cooldown guard on candidate selection:** In `UpdateProcAlert`, the candidate selection loop must also call `IsSpellAvailable(spellID)` and only accept the candidate if it returns `true`. A proc spell on cooldown must be skipped even if its overlay is active.
- [x] **AC2 ÔÇö No show when proc spell is on cooldown:** While `IsSpellAvailable` returns false for all candidates, `procAlertFrame` must remain hidden.
- [x] **AC3 ÔÇö Show resumes when cooldown expires:** On the next `Refresh` cycle after the spell comes off cooldown (`IsSpellAvailable` returns `true`), the proc icon must appear if the overlay is still active.
- [x] **AC4 ÔÇö No new taint risk:** `IsSpellAvailable` is already pcall-guarded. No additional taint exposure is introduced.
- [ ] **AC5 ÔÇö In-game verification:** Trigger a proc on a cooldown spell, wait for it to go on cooldown ÔÇö proc icon must disappear. When cooldown expires and overlay is still active, icon must reappear.

## Implementation Notes

**File:** `HekiLight.lua`
**Function:** `UpdateProcAlert` (~line 1683, after Story 9.1 change)

Extend the guard from Story 9.1:
```lua
for spellID in pairs(candidates) do
    if spellID ~= primarySpellID and C_SpellActivationOverlay.IsSpellOverlayed(spellID) then
        if GetSpellKeybind(spellID) ~= "" and IsSpellAvailable(spellID) then
            alertSpellID = spellID
            break
        end
    end
end
```

`IsSpellAvailable` is already a module-level local defined at ~line 1300. No new imports or declarations needed.
