# Story 5.2: IsActionInRange pcall Guard

**Status:** Done
**Epic:** 5 — Correctness Hardening
**Version:** v0.4.0 (TBD)
**Priority:** Implement first — fully independent, lowest risk
**Complexity:** XS (1–2 line change; wrapping an existing call)

## Context

`C_ActionBar.IsActionInRange(slotID)` is a taint-sensitive WoW API. Every other taint-sensitive call in `HekiLight.lua` is wrapped in `pcall()`. The call at approximately line 1743 inside `Refresh` is the sole exception — it is called bare on `rslot` (a realSlotID derived from `FindSpellActionButtons`). If the slot type is unexpected (e.g., a pet bar slot, a vehicle slot, or a slot returned under unusual circumstances), this call can taint the addon, silently breaking ability usage in combat.

## Acceptance Criteria

- [ ] `C_ActionBar.IsActionInRange(rslot)` in `Refresh` is wrapped in `pcall`
- [ ] On pcall failure, range tint is silently skipped (no error, no log spam)
- [ ] On pcall success, range tint behavior is identical to the current implementation
- [ ] A single `DLog("SLOT", ...)` entry fires if pcall fails (for diagnostics, not on every tick)

## Implementation Notes

- Locate the bare `IsActionInRange` call in `Refresh` — it follows the `rslot` lookup from `SLOT_BINDINGS`
- Replace with: `local ok, inRange = pcall(C_ActionBar.IsActionInRange, rslot)`
- Guard the tint logic: `if ok and inRange == false then ... end`
- The change-detection guard pattern already used elsewhere (`lastSlotSpellID`) can gate the DLog entry to fire only on the first failure per spell, not every frame

## Scope

**IN:** pcall wrap around `IsActionInRange` in `Refresh`; DLog on first failure per spell.
**OUT:** No changes to range tint logic, no changes to `GetRealSlot`, no changes to `queueCache`.

## Business Value

Eliminates the last unguarded taint-sensitive call in `HekiLight.lua`. A taint here silently breaks the player's ability to use spells in combat — the most severe possible addon failure mode.

## Risks

- **Low:** The pcall changes the error surface from a Lua runtime error to a silent skip. If `inRange` was previously returning an unexpected type, the new guard hides it. Mitigated by the DLog on failure.

## Files Changed

- `HekiLight.lua` — pcall wrapper around `IsActionInRange` in `Refresh`
