# Story 9.3: Proc Slot — Fix Cooldown Gate & Out-of-Combat Leak

**Epic:** EPIC-9 — Proc Slot Reliability
**Status:** Done
**Priority:** High

## User Story

As a Warlock player, I want the proc alert to show during combat whenever a proc fires (even if the spell is GCD-locked), and to never show outside of combat.

## Background

Two regressions introduced or exposed by EPIC-9:

### Bug A — Proc not showing during combat (`IsSpellAvailable` too strict)

Story 9.2 added `IsSpellAvailable(spellID)` to the candidate gate in `UpdateProcAlert`. `IsSpellAvailable` uses the secret-number pcall trick on `C_Spell.GetSpellCooldown`:

- `duration == 0` → plain 0 → true (off cooldown)
- `duration > 0` → secret number → pcall throws → false

For proc spells with no real cooldown (GCD-only), `C_Spell.GetSpellCooldown` may return a non-zero duration (the GCD remaining time) immediately after the player casts, causing `IsSpellAvailable` to return false. The proc fires exactly when the player is on GCD — the worst possible time for this check to block.

**Correct tool:** `not IsSpellOnCooldown(spellID)` (already exists, ~line 1316). `IsSpellOnCooldown` only tracks spells that have `GetSpellBaseCooldown > 1500ms` (entries added by `UNIT_SPELLCAST_SUCCEEDED`). GCD-only spells are never in `recentlyCastSpells` → always pass. Spells on a real CD (just cast with CD > 1.5s) correctly block.

### Bug B — Proc showing outside combat (no combat gate on event handler)

`SPELL_ACTIVATION_OVERLAY_GLOW_SHOW` calls `UpdateProcAlert(currentSuggestionID)` directly (~line 2126), with no check against combat state. Outside of combat, `currentSuggestionID` is nil, so `spellID ~= nil` is trivially true — the proc frame shows if a spell has an active overlay + keybind + passes availability.

**Fix:** Gate the direct event-handler calls to `UpdateProcAlert` with `inCombat` (or the equivalent `db.showMode == "always"` for users who prefer always-on mode).

## Acceptance Criteria

- [x] **AC1 — Replace `IsSpellAvailable` with `IsSpellOnCooldown` in proc candidate gate:** In `UpdateProcAlert` (~line 1685), replace `and IsSpellAvailable(spellID)` with `and not IsSpellOnCooldown(spellID)`. GCD-only proc spells must pass this check immediately when the proc fires.
- [x] **AC2 — Real-cooldown spells still suppressed:** A proc spell that the player just cast and which has a real cooldown (`GetSpellBaseCooldown > 1500ms`) must remain hidden until `IsSpellAvailable` clears it from `recentlyCastSpells`.
- [x] **AC3 — Combat gate on event-triggered `UpdateProcAlert`:** In the `SPELL_ACTIVATION_OVERLAY_GLOW_SHOW` and `SPELL_ACTIVATION_OVERLAY_GLOW_HIDE` event handlers (~lines 2126, 2136), only call `UpdateProcAlert` when `inCombat == true` OR `db.showMode == "always"`. Outside combat with `showMode == "active"`, the direct event calls must be no-ops.
- [x] **AC4 — Proc clears on GLOW_HIDE outside combat:** If the proc alert is showing (e.g. showMode == "always") and the glow ends, the frame must still hide via the normal hide-path.
- [x] **AC5 — In-game verification (Warlock):** Trigger a Warlock proc during combat — proc icon must appear promptly. Log out of combat — proc icon must not appear outside combat.

## Implementation Notes

**File:** `HekiLight.lua`

### Fix 1 — `UpdateProcAlert` candidate loop (~line 1685)

Change:
```lua
if GetSpellKeybind(spellID) ~= "" and IsSpellAvailable(spellID) then
```
to:
```lua
if GetSpellKeybind(spellID) ~= "" and not IsSpellOnCooldown(spellID) then
```

`IsSpellOnCooldown` is a module-level local defined at ~line 1316. It returns `true` only when the spell is in `recentlyCastSpells` AND `IsSpellAvailable` has not cleared it yet.

### Fix 2 — Event handler combat gate (~lines 2126, 2136)

Change both `UpdateProcAlert` calls in the GLOW_SHOW / GLOW_HIDE handlers from:
```lua
UpdateProcAlert(currentSuggestionID)
```
to:
```lua
if inCombat or db.showMode == "always" then
    UpdateProcAlert(currentSuggestionID)
end
```
