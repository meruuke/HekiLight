## User Story

As a player with the same spell on multiple action bars, I want to pin which action bar slot HekiLight uses for that spell's keybind label, so I always see the binding I actually press.

## Background

`GetSpellKeybind(spellID)` iterates `C_ActionBar.FindSpellActionButtons(spellID)` and returns the keybind of the **first non-RA slot found**. The API return order is not guaranteed to match the player's mental model. A player who has Frostbolt on both slot 1 ("1") and slot 37 ("F") may consistently see "F" when they expect "1".

This story adds:
1. A per-character override store: `dbChar.keybindOverrides[spellID] = slotID`
2. A discovery command to list all bar slots and their bindings for a spell
3. Pin and clear slash commands
4. Modified `GetSpellKeybind` that checks the override before iterating all slots

The override stores the **slot ID** (not the key string) so the displayed keybind auto-updates if the player rebinds that slot later.

## Acceptance Criteria

- [ ] **AC1 ‚Äî Override store initialised:** `InitDB()` initialises `HekiLightDBChar.keybindOverrides` as an empty table if missing. Existing saves are not overwritten.
- [ ] **AC2 ‚Äî `/hkl kb slots <spellID>` command:** Prints all non-RA bar slots containing the spell, each with its current keybind (or `(unbound)` if empty). Example output:
  ```
  [HekiLight] Keybinds for spellID 116 (Frostbolt):
    Slot 1  ‚Üí 1
    Slot 37 ‚Üí F
    Slot 73 ‚Üí (unbound)
  ```
  If the spell has no bar slots, prints `No action bar slots found for spellID 116`.
- [ ] **AC3 ‚Äî `/hkl kb pin <spellID> <slot>` command:** Sets `dbChar.keybindOverrides[spellID] = slot`. Prints confirmation including the resolved keybind at that slot. Rejects invalid input (non-numeric spellID or slot, slot ‚â§ 0) with a clear error message. Does **not** require the spell to currently be in that slot ‚Äî the override is stored regardless (graceful no-op if slot is empty at render time).
- [ ] **AC4 ‚Äî `/hkl kb clear <spellID>` command:** Removes `dbChar.keybindOverrides[spellID]`. Prints confirmation. Silently succeeds if no override existed.
- [ ] **AC5 ‚Äî `/hkl kb list` command:** Prints all current overrides with spell name (via `C_Spell.GetSpellInfo`) and the pinned slot. Prints `No keybind overrides set.` if the table is empty.
- [ ] **AC6 ‚Äî `GetSpellKeybind` respects override:** Before iterating `FindSpellActionButtons`, check `dbChar.keybindOverrides[spellID]`. If a slot ID is stored, call `GetSlotKeybind(overrideSlot)`. If the result is non-empty, return it (and update `keybindCache`). If the result is empty (spell removed from that slot), fall through to the normal multi-slot scan.
- [ ] **AC7 ‚Äî Cache coherence:** When the override slot resolves a keybind, populate `keybindCache[spellID]` as normal so the transient-miss fallback still works.
- [ ] **AC8 ‚Äî DLog tagging:** When the override path is taken, log `DLog("KEYBIND", "spellID=%d key=%s source=override slot=%d")`. When the override is present but resolves empty (fallthrough), log `source=override_miss`.
- [ ] **AC9 ‚Äî In-game verification:** With a spell present on two slots with different bindings, `/hkl kb pin` to the non-default slot and confirm the label in the overlay updates on next render without `/reload`.

**Override check inside `GetSpellKeybind`:**
```lua
local overrideSlot = dbChar.keybindOverrides and dbChar.keybindOverrides[spellID]
if overrideSlot then
    local key = GetSlotKeybind(overrideSlot)
    if key ~= "" then
        keybindCache[spellID] = key
        DLog("KEYBIND", string.format("spellID=%d key=%s source=override slot=%d", spellID, key, overrideSlot))
        return key
    end
    DLog("KEYBIND", string.format("spellID=%d source=override_miss slot=%d", spellID, overrideSlot))
    -- fall through to normal scan
end
```

## Business Value

Players who bind the same spell on multiple bars (primary bar + class bar, or primary bar + multi-page layout) see a keybind that doesn't match the key they actually press. This breaks the core promise of the keybind label feature ‚Äî reducing lookup friction during combat. The override gives users authoritative control at near-zero implementation cost (no new frames, no events).

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| Pinned slot not in `SLOT_BINDINGS` (addon bar type not mapped by `RebuildSlotBindings`) | Low | Low | `GetSlotKeybind` returns `""` ‚Üí override falls through to normal scan. DLog `source=override_miss`. Player can re-pin to a supported slot. |
| `FindSpellActionButtons` transient nil at combat start | Existing | Low | Override path calls `GetSlotKeybind(overrideSlot)` directly ‚Äî bypasses `FindSpellActionButtons` entirely. The override path is *more* robust than the default path at combat start. |
