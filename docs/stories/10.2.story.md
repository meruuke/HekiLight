## User Story

As a player who sees the wrong keybind label for a rotation spell, I want to pin the correct bar slot directly from the Settings panel — without needing to know the spellID or use slash commands.

## Background

Story 10.1 delivers full slash-command control (`/hkl kb pin`, `clear`, `list`). Story 10.2 makes the feature discoverable and visual by extending the **existing Ignored Spells collapsed section** inside `BuildSettingsPanel` (~line 700).

The existing spell dropdown (`ignoreDD`) is already populated live from `C_AssistedCombat.GetRotationSpells()` — only spells currently in the player's rotation appear there. That is exactly the right scope: the keybind problem only affects spells that are showing in the overlay. Reusing this dropdown avoids building a new panel and keeps all per-spell preferences in one place.

### Two-step flow (UI)

**Step 1 — Spell selection (existing):** Player selects a spell from the existing dropdown. `selectedIgnoreSpellID` is set. Both the existing "Add to ignore list" button and the new slot-picker controls become active simultaneously.

**Step 2 — Slot selection (new):** A second dropdown (`slotDD`), positioned below the first row, is populated with all non-RA bar slots for the selected spell (via `C_ActionBar.FindSpellActionButtons`). Each entry shows `Slot N → Key` (or `(unbound)`). Player selects a slot and clicks "Pin keybind". `dbChar.keybindOverrides[spellID] = slotID` is set, and the override list refreshes.

### Layout within the Ignored Spells section

```
[Description text]
[ignoreDD: Select spell...]  [Add to ignore list]
[slotDD: Select slot...]     [Pin keybind btn]     ← row 2, hidden until spell selected w/ ≥2 slots
─── Ignored ───
  [icon] Spell Name [Remove]
─── Keybind Overrides ───
  [icon] Spell Name  Slot 37 → F  [Clear]
```

The `slotDD` row (row 2) is **hidden by default** and shown only after a spell is selected in `ignoreDD` AND `FindSpellActionButtons` returns at least two non-RA slots for that spell.

## Acceptance Criteria

- [ ] **AC1 — Slot picker row hidden by default:** At panel open, the slot dropdown (`slotDD`) row and its "Pin keybind" button are hidden. They become visible only after a spell with ≥2 non-RA bar slots is selected from `ignoreDD`. If the selected spell has 0 or 1 non-RA slots, the slot picker row remains hidden and a small label reads `Only one bar slot found — no override needed.`
- [ ] **AC2 — Slot dropdown populated on spell selection:** When a spell is selected in `ignoreDD`, `UIDropDownMenu_Initialize` for `slotDD` is called with entries built from `C_ActionBar.FindSpellActionButtons(selectedIgnoreSpellID)`, filtering out RA slots. Each entry text: `Slot N → Key` (using `GetSlotKeybind(slot)`) or `Slot N → (unbound)`. The currently pinned slot (if any) is marked `[pinned]`.
- [ ] **AC3 — "Pin keybind" button sets override:** Clicking "Pin keybind" calls the same logic as `/hkl kb pin <spellID> <slot>` — sets `dbChar.keybindOverrides[selectedIgnoreSpellID] = selectedSlotID`, prints a confirmation to chat, and calls `RefreshOverrideList()`.
- [ ] **AC4 — Keybind overrides sub-list rendered:** Below the existing ignored-spell rows, a separator label "Keybind Overrides" introduces a second row pool. Each override row shows: spell icon (16×16), spell name, pinned slot number, resolved keybind (or `(unbound)`), and a "Clear" button. Empty state label: `No keybind overrides set.`
- [ ] **AC5 — "Clear" button removes override:** Clicking "Clear" on an override row calls the same logic as `/hkl kb clear <spellID>` — removes the entry from `dbChar.keybindOverrides`, hides the row, repositions remaining override rows. Slot picker row resets if the cleared spell is currently selected.
- [ ] **AC6 — Section expansion refreshes both lists:** The section's `OnClick` expansion handler (already calls `RefreshIgnoreList()`) also calls `RefreshOverrideList()`. Both lists reflect current `dbChar` state on every expand.
- [ ] **AC7 — No new subpanel registered:** `Settings.RegisterCanvasLayoutSubcategory` is NOT called. All new frames are children of the existing `ignoreBody` frame.
- [ ] **AC8 — No taint:** All new frames created at `ADDON_LOADED`. "Pin" and "Clear" buttons use `SetScript("OnClick", ...)` with no protected frame ancestry. `FindSpellActionButtons` call in the dropdown initializer is pcall-guarded.
- [ ] **AC9 — In-game verification:** Select a spell with two different keybinds in the dropdown → slot picker row appears, shows both slots with their keys → select the non-default slot → click "Pin keybind" → override row appears in the list → overlay keybind label updates on next render cycle without `/reload`.

**Existing patterns to follow exactly:**
```lua
-- Row pre-allocation pattern (from RefreshIgnoreList)
local overrideRowPool = {}
-- ... pre-create rows at ADDON_LOADED or lazily ...
RefreshOverrideList = function()
    for _, row in ipairs(overrideRowPool) do row:Hide() end
    local sorted = {}
    for sid in pairs(dbChar.keybindOverrides) do sorted[#sorted+1] = sid end
    table.sort(sorted)
    -- ... show rows ...
end

-- selectedSlotID tracking (mirrors selectedIgnoreSpellID)
local selectedSlotID = nil
UIDropDownMenu_Initialize(slotDD, function(self, level)
    local slots = C_ActionBar.FindSpellActionButtons(selectedIgnoreSpellID)
    if slots then
        for _, slot in ipairs(slots) do
            if not C_ActionBar.IsAssistedCombatAction(slot) then
                local info = UIDropDownMenu_CreateInfo()
                local key = GetSlotKeybind(slot)
                info.text = "Slot " .. slot .. " → " .. (key ~= "" and key or "(unbound)")
                    .. (dbChar.keybindOverrides[selectedIgnoreSpellID] == slot and " [pinned]" or "")
                info.func = function() selectedSlotID = slot end
                UIDropDownMenu_AddButton(info)
            end
        end
    end
end)
```

**Height calculation note:** After both lists, total height = `math.abs(ignoreListBaseY) + (ignoreRowCount + overrideRowCount + 1) * 30 + 20` — the `+1` accounts for the "Keybind Overrides" separator label row.

## Business Value

Players can fix an incorrect keybind label in three clicks inside a panel they already know, using spell names instead of spellIDs. No new Settings pages to discover. Per-spell preferences stay in one place.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| `FindSpellActionButtons` returns nil transiently when dropdown opens | Low | Low | pcall-guard the call; if nil, populate slot dropdown with a single disabled entry: `(bar slots not yet loaded — try again)` |
| Section height grows and clips at the bottom of the Settings canvas | Low | Medium | Measure existing section height; override list uses the same 30px-per-row rhythm as the ignore list — cap at 20 rows with a `+N more` footer |
| Shared `selectedIgnoreSpellID` state creates confusion (user selects spell, does ignore action, slot picker still shows) | Low | Low | Reset `selectedIgnoreSpellID` and hide slot picker row after either "Add to ignore list" or "Pin keybind" is clicked |
