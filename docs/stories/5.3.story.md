# Story 5.3: GetRealSlot for Secondary Spells

**Status:** Done
**Epic:** 5 ŌĆö Correctness Hardening
**Version:** v0.4.0 (TBD)
**Priority:** Implement second ŌĆö prerequisite for Story 5.1
**Complexity:** M (extends pre-allocated struct; new resolution path in `GetSuggestionQueue`)

## Context

`GetSuggestionQueue` builds secondary slots from `C_AssistedCombat.GetRotationSpells()`, which returns **base spellIDs** (e.g., Death and Decay `43265`). The primary pipeline calls `GetRealSlot(spellID)` to find a regular action bar slot and resolve the override icon via `GetActionInfo`. The secondary pipeline currently skips `GetRealSlot` ŌĆö it only tracks the base spellID. This means:

1. Secondary slot icons show the base spell texture, not the talent-overridden one (e.g., Death and Decay icon instead of Defile icon)
2. `FindSpellActionButtons(baseID)` may return nil if the bar slot is registered under the override spellID, silently dropping the real slot reference

This story makes `GetRealSlot` part of the secondary spell resolution path, mirroring the primary pipeline.

## Acceptance Criteria

- [ ] For each secondary spell `sid` from `GetRotationSpells()`, `GetRealSlot(sid)` is called (pcall-guarded)
- [ ] If `GetRealSlot` succeeds, the returned `realSlotID` is stored in `queueCache` alongside `sid`
- [ ] `GetActionInfo(realSlotID)` is used to resolve the override icon for secondary slots (same as primary)
- [ ] If `GetRealSlot` returns nil, the slot falls back to `C_Spell.GetSpellInfo(sid)` icon (current behavior, graceful degradation)
- [ ] Every override substitution on a secondary slot fires a `DLog("OVERRIDE", ...)` entry
- [ ] `queueCache` entries store `{ spellID, overrideSpellID, realSlotID, onCooldown }` ŌĆö `overrideSpellID` is new

## Implementation Notes

- `queueCache` is a pre-allocated module-level table; add `overrideSpellID` and `realSlotID` fields to each entry (initialized to `false` at startup to avoid per-frame allocation)
- `GetRealSlot(sid)` already wraps `FindSpellActionButtons` in pcall; nil return is the normal failure case for unlearned or override-indexed spells
- If `GetRealSlot(sid)` returns nil, try `GetRealSlot` with the override ID obtained from `GetActionInfo` of any found slot ŌĆö but only if a slot was found at all; do not spin on failure
- The icon rendered for each secondary slot should prefer `GetActionInfo(realSlotID).icon` over `C_Spell.GetSpellInfo(sid).iconID` when a realSlotID is available

## Scope

**IN:** Extend `queueCache` struct with `overrideSpellID` and `realSlotID` fields; call `GetRealSlot` per secondary spell; resolve override icon via `GetActionInfo`; fallback to base icon if nil.
**OUT:** No changes to the primary spell pipeline (`GetActiveSuggestion`). No changes to `Refresh` range logic. No changes to keybind lookup.

## Business Value

Secondary slots will show the correct talent-override icon (Defile instead of Death and Decay, etc.), making the suggestion strip accurate for talent-heavy specs. Also lays the foundation for correct deduplication in Story 5.1.

## Risks

- **Medium:** `queueCache` is pre-allocated at startup. The new `overrideSpellID`/`realSlotID` fields must be explicitly initialized to `false` in the pre-allocation block ŌĆö omitting this causes `nil` access on the first tick before a slot is filled.
- **Low:** `GetActionInfo` is pcall-guarded in the primary pipeline; the secondary path must maintain the same guard. Double-wrapping (pcall inside pcall) is harmless but should be avoided for clarity.

## Files Changed

- `HekiLight.lua` ŌĆö `queueCache` struct extended; secondary spell resolution in `GetSuggestionQueue`
